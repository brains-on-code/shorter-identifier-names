How do we read source code?

Participate in our study!

Source code is often written and then not looked at for considerable lengths of time. It needs to be written so that a programmer coming to it after such a long time is able to quickly understand it and make appropriate changes. Writing code that is understandable, without the context it was written with, is a difficult task.

Programmers often feel that code is bad or good, but can this qualitative feeling be measured in a quantative way? In order to research this question, I developed this experiment.
